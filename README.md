## 🌍 What is Campus OS?

**Campus OS** is not a student portal. It is the **central nervous system of a modern university** — an all-in-one, AI-powered platform that transforms how students learn, how administrators operate, and how campuses function.

Built at the intersection of **generative AI, gamification psychology, and operational intelligence**, Campus OS reimagines every touchpoint of campus life:

- A **student files a grievance** → AI classifies, routes, and escalates it automatically
- A **student loses their laptop** → AI semantically matches it with a found item in seconds
- A **student asks about exam policy** → AI answers from the actual PDF in plain English
- A **student walks into the library** → their RPG character gains XP and Intelligence stats

This is education, **leveled up**.

### 🔄 LangGraph Agentic Workflows

The server runs **autonomous AI agents** via LangGraph that operate continuously:

```typescript
// Autonomous agent loop — running 24/7
const campusAgent = {
  analyze_grievances()    // → Detect critical/duplicate complaints
  escalate_ticket()       // → Auto-escalate overdue SLA tickets  
  detect_patterns()       // → Identify recurring maintenance failures
  match_items()           // → Semantic lost/found matching
  send_notification()     // → Push WhatsApp/email alerts
}
```

These agents respond via **Server-Sent Events (SSE)**, streaming real-time reasoning directly to the admin dashboard.

## 🏛️ Campus Operations Modules

### 📢 Module 1 — AI Grievance Triage System

The most intelligent complaint management system ever built for a campus.

**How it works:**
1. Student submits a grievance with free-form description
2. GPT-4 instantly classifies category, urgency (1–5), and sentiment
3. Auto-routes to correct department (IT / Facilities / Admin / Hostel)
4. Duplicate detection prevents ticket flooding
5. SLA clock starts — escalation risk score calculated every hour
6. Admin sees live Kanban board with color-coded priorities

```
GRIEVANCE LIFECYCLE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  [NEW] ──► [ASSIGNED] ──► [IN PROGRESS] ──► [RESOLVED]
              │                                    ▲
              └──► [ESCALATED] ─────────────────────┘
                   (Triggered by SLA breach or Critical Risk Score)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**Key capabilities:**
- ✅ AI-powered auto-categorization across 8+ departments
- ✅ Escalation risk score (0–100) with real-time recalculation
- ✅ Sentiment analysis (frustrated / neutral / angry / urgent)
- ✅ Duplicate detection — prevents ticket flooding
- ✅ WhatsApp notifications via Twilio for critical issues
- ✅ Full audit trail with SLA tracking
- ✅ Admin Kanban board with drag-and-drop status updates

### 📖 Module 3 — AI Policy Navigator (RAG Engine)

**Never read a 200-page PDF again.** Ask anything, get a precise answer instantly.

**How it works:**
1. Admin uploads official PDF documents (hostel rules, exam policies, fee structures)
2. `pdfjs-dist` extracts full text content page-by-page
3. Documents are stored and indexed in the active policy context
4. Students ask questions in plain English
5. Azure GPT-4 answers using **only** the uploaded documents (hallucination-protected)

### 🔍 Module 4 — AI Lost & Found Matcher

**Semantic intelligence that reunites students with their belongings.**

**How it works:**
1. Student reports lost item with description, photo, and last known location
2. Finder logs found item with photo through the found item portal
3. AI semantic matcher evaluates all active lost/found pairs
4. Confidence score calculated based on category, description, color, location
5. Automatic notification sent to both parties when match ≥ 90%
6. Secure OTP generated for verified item claiming

```

## 🚀 Quick Start

### Prerequisites

```bash
node --version   # >= 18.0.0
npm --version    # >= 9.0.0
```

### 1. Clone & Install

```bash
git clone https://github.com/Aditya-Kumrawat/Campus-OS.git
cd Campus-OS
npm install
```

### 2. Configure Environment

```bash
cp env .env
```

Open `.env` and fill in your API keys (see [Environment Configuration](#-environment-configuration) below).

### 3. Run Development Server

```bash
npm run dev
```

🌐 Open **[http://localhost:5173](http://localhost:5173)**

### 4. Build for Production

```bash
npm run build    # Builds frontend (dist/spa) + backend (dist/server)
npm start        # Starts production server on :3000
```

### 5. Docker Deployment

```bash
docker build -t campus-os .
docker run -p 3000:3000 --env-file .env campus-os
```

## 👥 Roles & Access Control

Campus OS implements a three-tier role system with route-level protection:

| Role | Access | Demo Credentials |
|------|--------|-----------------|
| 🎓 **Student** | Personal dashboard, grievance filing, lost/found, policy Q&A, attendance, canteen, scholarships | `student@campus.edu` / `password123` |
| 👨‍🏫 **Teacher** | All student features + course management, test creation, attendance management | `teacher@campus.edu` / `teacher123` |
| 🛡️ **Admin** | All features + administrative dashboards, policy uploads, grievance management, system oversight | `admin@campus.edu` / `admin123` |

All protected routes are wrapped in `<ProtectedRoute>` with role validation — unauthorized access redirects to login.

## 📡 API Reference

### Core Endpoints

```
GET  /api/ping                         Health check

POST /api/campus-agent                 LangGraph agent (SSE streaming)
     Body: { message: string, context?: object }
     Returns: Server-Sent Events stream

POST /api/pdf/process                  Upload + extract PDF text
     Body: multipart/form-data (file: PDF)
     Returns: { pages: number, text: string, metadata: object }

POST /api/whatsapp/send                Send WhatsApp notification
     Body: { to: string, message: string }
     Returns: { success: boolean, sid: string }

POST /api/vapi/call                    Create voice AI call
     Body: { phoneNumber: string, assistantId: string }
     Returns: { callId: string }

POST /api/visualization/generate       Generate chart/heatmap data
     Body: { type: string, data: array }
     Returns: { chartData: object }
```

### LangGraph Agent Tools

| Tool | Description | Trigger |
|------|-------------|---------|
| `analyze_grievances()` | Scan for critical/duplicate complaints | Every 15 min |
| `escalate_ticket(id)` | Promote ticket priority + notify admin | SLA breach |
| `detect_patterns()` | Find recurring maintenance failures | Hourly |
| `match_items()` | Semantic lost/found pair evaluation | On new report |
| `send_notification(msg)` | Push WhatsApp/email alert | Agent decision |
