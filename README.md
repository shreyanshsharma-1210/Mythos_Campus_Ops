## 🎯 The Sovereign System: Why CampusOS?

In the context of a student’s life at a competitive university, the "problem" isn't just a lack of productivity—it's a lack of **engagement and psychological reward.**

Here is the breakdown of the core problems **The Sovereign System** is designed to solve:

### **1. The "Academic Void" (Lack of Real-Time Feedback)**

In a video game, you kill a monster and immediately see your XP bar move. In college, you study for 10 hours and feel nothing until a mid-term exam months later.

* **The Problem:** The "feedback loop" of traditional education is too slow, leading to burnout and procrastination.
* **Our Solution:** The **Hunter’s Hub** turns every hour of study into immediate, visible XP and attribute growth on your Radar Chart.

### **2. The "Resume Paradox" (Invisible Skills)**

Students spend years building skills (Python, Public Speaking, Leadership), but these remain "invisible" or buried in a flat, boring PDF resume.

* **The Problem:** Traditional resumes fail to show the *effort* and *consistency* behind a skill.
* **Our Solution:** **The Hunter’s Vault** digitizes skills into "Artifacts." You don't just "know" Python; you possess the *S-Rank Python Dagger*, earned through verified "Dungeon Raids."

### **3. The "Canteen Trap" (Energy & Time Leakage)**

Campuses are full of "Danger Zones"—places like canteens or gaming lounges where students unintentionally spend hours losing focus and energy.

* **The Problem:** Students lack spatial awareness of how their environment affects their productivity.
* **Our Solution:** The **AR World Map** highlights these locations as "Energy Drain Zones," using geofencing to warn the user when their "Mana" is being depleted by staying in a non-productive area for too long.

### **4. The "Solo Grind" Isolation**

Despite being surrounded by thousands of peers, students often feel they are grinding through their degree alone.

* **The Problem:** Networking feels like a chore or a formal event rather than a natural interaction.
* **Our Solution:** The **NFC Social Link** and **Guild Hall** turn networking into a "Party System." Finding a study partner isn't a "meeting"; it’s a "Raid" where both players get a synergy buff.

### **5. Mental Fatigue & The "Mana" Depletion**

The high-pressure environment leads to mental exhaustion, but students often ignore it until they crash.

* **The Problem:** A lack of structured recovery and mental wellness tracking.
* **Our Solution:** The **Mana Recovery** pillar and **System Oracle** monitor your fatigue levels. The system mandates "Recovery Quests" (meditation/sleep) to clear the "Fatigue Debuff," treating mental health as a core game mechanic.

### **Problem-Solution Summary Table**

| The Pain Point | The Sovereign Solution | Gamified Concept |
| --- | --- | --- |
| **Boredom** | Real-time attribute tracking | **Solo Leveling XP** |
| **Disorganization** | Daily segregated quest grid | **Quest Matrix** |
| **Distraction** | Geofenced campus warnings | **Dungeon vs. Danger Zones** |
| **Loneliness** | NFC-based party building | **Guild Synergy** |
| **Burnout** | Mandatory recovery sessions | **Mana Restoration** |

## ✨ Core Features

### ⚔️ The Hunter's System (Gamified Interaction)

CampusOS transforms the educational experience into an RPG world where learning powers up your character.

<div align="center">

| Attribute | Description |
|-----------|-------------|
| **🏰 The Hunter’s Hub**<br/>*(3D Dashboard)* | The central command center featuring your high-fidelity 3D Anime Character. Visualization of your **Hunter Rank (E to S)** and a Radar Stat Chart for **[STR]**, **[INT]**, **[WIL]**, and **[SOC]**. |
| **📊 The Quest Matrix**<br/>*(Pillared Tasks)* | A 2x2 grid segregating life into RPG pillars: **[STR] Physical Dungeon**, **[INT] Arcane Knowledge**, **[WIL] Mana Recovery**, and **[SOC] Hunter’s Guild**. |
| **🗺️ AR World Map**<br/>*(Campus Dungeons)* | Transforms your university into a game world. Libraries are **Dungeons** (High XP), Canteens are **Danger Zones** (Energy Depletion). |
| **💪 Physical Dungeon**<br/>*(Google Fit Sync)* | Automated XP gain through health APIs. Steps, workouts, and calories directly power up your **[STR]** attribute. |
| **🤝 NFC Social Link**<br/>*(The Handshake)* | Tap phones to trigger a **'Synergy Duel'** trivia mini-game. Completing social quests grants temporary **XP Multiplier Buffs**. |
| **🎒 The Hunter’s Vault**<br/>*(Inventory/Skills)* | Your resume reimagined. Skills (e.g., Python) are stored as **Artifacts** in a branching **Skill Tree**. |
| **🔮 System Oracle**<br/>*(AI Mentor)* | A **Gemini-powered** AI guide. Uses **'Shadow Extraction'** to create personalized AI tutors based on your learning history. |
| **⚡ Mana Rifts**<br/>*(AR Tappies)* | Hidden energy spots scattered across campus. Find and tap them for free **Mana** or **'Focus Buffs'**. |
| **🏆 The Campus Cup**<br/>*(Guild Wars)* | Department-based guilds (e.g., House of Engineering). XP contributions fuel the Guild Total for monthly rewards. |
| **📜 The Bounty Board**<br/>*(Community Feed)* | Crowdsourced task board. Complete campus bounties to earn **Karma Points** and unique **Social Artifacts**. |

## 🚀 Quick Start

### Prerequisites

<p align="center">
<img src="https://img.shields.io/badge/Node.js-≥18.x-339933?style=for-the-badge&logo=node.js&logoColor=white" />
<img src="https://img.shields.io/badge/npm-≥9.x-CB3837?style=for-the-badge&logo=npm&logoColor=white" />

</p>

### Installation & Setup

```bash
# 1️⃣ Clone the repository
git clone https://github.com/Aditya-Kumrawat/CampusOS.git
cd campusos

# 2️⃣ Install dependencies
npm install

# 3️⃣ Setup environment variables
cp .env.example .env
# Edit .env with your configuration:
# - Firebase credentials  
# - VAPI API key
# - Other service credentials

# 4️⃣ Install Python dependencies (for AI proctoring)
cd ai_proctoring_module
pip install -r requirements.txt
cd ..
```

### Development Mode

```bash
# Terminal 1: Start unified dev server (Frontend + Backend)
npm run dev
# Frontend: http://localhost:5173
# Backend API: http://localhost:5173/api/*

# Terminal 2: Start AI proctoring service (optional)
cd ai_proctoring_module
python proctoring_suite.py
# AI Service: http://localhost:8000
```

### Production Build

```bash
# Build both client and server
npm run build

# Start production server
npm start
# Server runs on: http://localhost:3000
# Serves both API and static SPA
```

## 📚 API Documentation

CampusOS provides a comprehensive RESTful API for developers to integrate with the platform. Below are some key endpoints:

### Authentication

```http
POST /api/auth/login
Content-Type: application/json

{
  "email": "student@example.com",
  "password": "securepassword123"
}
```

### Key Endpoints

| Endpoint | Method | Description | Authentication |
|----------|--------|-------------|----------------|
| `/api/courses` | GET | List all available courses | Bearer Token |
| `/api/courses/:id` | GET | Get course details | Bearer Token |
| `/api/tests` | POST | Create a new test | Teacher Token |
| `/api/tests/:id/submit` | POST | Submit test answers | Student Token |
| `/api/analytics/student/:id` | GET | Get student analytics | Bearer Token |
| `/api/ai/proctor/start` | POST | Start AI proctoring session | Student Token |

### WebSocket Connections

```javascript
const socket = new WebSocket('wss://api.campusos.com/ws/proctoring');

};
```

   - Implement role-based access controls
   - Regularly review and update permissions

   - Keep all systems patched and updated
   - Use Web Application Firewall (WAF)
   - Implement rate limiting and DDoS protection

3. **Data Protection**
   - Regular data backup and recovery testing
   - Encryption of sensitive data at rest
   - Secure API key management

### For Educators

1. **Secure Content**
   - Use secure file sharing methods
   - Be cautious with third-party integrations
   - Regularly review shared materials

2. **Student Privacy**
   - Share student data only through secure channels
   - Be mindful of recording settings in virtual classrooms
   - Follow school/district privacy policies

1. **Account Security**
   - Use strong, unique passwords
   - Enable multi-factor authentication
   - Never share login credentials

   - Ensure a private, well-lit space for proctored exams
   - Close unnecessary applications during tests
   - Use a secure, private network connection
